# Book Management Web Application

این پروژه یک وب‌اپلیکیشن کامل و ساده برای مدیریت کتاب است که با استفاده از Flask، SQLite و یک ماژول پایتونی سفارشی ساخته شده است. هدف اصلی این پروژه تمرین توسعه وب، کار با دیتابیس و یادگیری ساختاردهی پروژه‌های پایتونی به صورت ماژولار است. این پروژه به کاربران امکان می‌دهد کتاب‌ها را اضافه کنند، لیست کتاب‌های موجود را مشاهده کنند و کتاب‌های دلخواه را حذف کنند. داده‌ها به گونه‌ای ذخیره می‌شوند که پس از بسته شدن برنامه نیز باقی بمانند و اطلاعات از بین نرود. طراحی ماژولار برنامه باعث شده بخش منطق مدیریت کتاب از بخش وب جدا باشد و در آینده توسعه برنامه و افزودن ویژگی‌های پیشرفته بسیار ساده‌تر شود.

ویژگی‌ها:
- اضافه کردن کتاب جدید با وارد کردن نام کتاب
- مشاهده لیست کامل کتاب‌های ذخیره‌شده به صورت مرتب
- حذف کتاب‌ها با یک کلیک ساده
- ذخیره دائمی داده‌ها با استفاده از SQLite برای جلوگیری از از بین رفتن اطلاعات
- مدیریت پیام‌ها و هشدارها در زمانی که هیچ کتابی در سیستم موجود نباشد
- طراحی ماژولار و جداسازی منطق مدیریت کتاب از بخش وب
- قابلیت توسعه و افزودن ویژگی‌های پیشرفته مانند جستجو، فیلتر، دسته‌بندی و سیستم کاربری
- قابلیت استفاده آسان و تجربه کاربری ساده با رابط کاربری روشن و قابل فهم

ساختار پروژه:
- **app.py**: فایل اصلی Flask که روت‌ها، دریافت و ارسال داده‌ها و اجرای برنامه را مدیریت می‌کند. این فایل روت‌های مختلف برای مشاهده لیست کتاب‌ها، اضافه کردن کتاب و حذف کتاب را مدیریت می‌کند و داده‌ها را بین فرم‌ها، دیتابیس و ماژول Library هماهنگ می‌کند.
- **Library/library.py**: شامل کلاس Library و توابع مدیریت کتاب‌ها است. این کلاس تمامی عملیات مربوط به کتاب‌ها را مدیریت می‌کند، مانند افزودن کتاب، حذف کتاب، جلوگیری از ورود کتاب‌های تکراری و مدیریت پیام‌ها در صورت نبود کتاب.
- **templates/index.html**: رابط کاربری شامل فرم ثبت کتاب و نمایش لیست کتاب‌هاست. این فایل طراحی ساده‌ای دارد تا کاربر بتواند به راحتی کتاب‌ها را اضافه یا حذف کند و پیام‌ها را مشاهده کند.
- **static/style.css**: شامل استایل‌های صفحه است که فرم، دکمه‌ها، لیست کتاب‌ها و پیام‌ها را به شکل زیبا و خوانا نمایش می‌دهد.
- **requirements.txt**: لیست پکیج‌های مورد نیاز برای اجرای پروژه مانند Flask و SQLite.
- **database.db**: دیتابیس SQLite که داده‌ها را به صورت دائمی ذخیره می‌کند و جدول اصلی آن `books` است.

ماژول Library:
کلاس Library هسته مدیریت کتاب‌ها در این پروژه است. توابع اصلی آن شامل:
- `add_book`: برای اضافه کردن کتاب جدید و جلوگیری از اضافه شدن کتاب‌های تکراری
- `delete_book`: حذف کتاب دلخواه از لیست و دیتابیس
- `get_books`: برگرداندن لیست کتاب‌ها برای نمایش در رابط کاربری
- `message_handler`: بررسی وضعیت کتاب‌ها و نمایش پیام مناسب در صورتی که هیچ کتابی موجود نباشد
این طراحی باعث می‌شود که منطق مدیریت کتاب‌ها به صورت مستقل از بخش وب باشد و توسعه و افزودن قابلیت‌های جدید مانند دسته‌بندی یا جستجو بسیار ساده شود. هر تغییر در ماژول Library به راحتی می‌تواند روی تمام بخش‌های برنامه اعمال شود بدون اینکه بخش وب تحت تأثیر قرار بگیرد.

دیتابیس SQLite:
دیتابیس پروژه شامل جدول `books` با ستون‌های `id` و `name` است. ستون `id` کلید اصلی و خودکار است و ستون `name` نام کتاب را ذخیره می‌کند. در اجرای اولیه برنامه، بررسی می‌شود که آیا جدول وجود دارد یا خیر و در صورت نبود، جدول ایجاد می‌شود. عملیات اضافه کردن، حذف و مشاهده کتاب‌ها همگی از طریق SQLite انجام می‌شوند تا داده‌ها بعد از بستن برنامه حفظ شوند. این روش باعث می‌شود برنامه بدون نیاز به سرور خارجی یا دیتابیس پیچیده کار کند و برای پروژه‌های کوچک و متوسط مناسب باشد.

روت‌ها و عملکرد برنامه:
- **صفحه اصلی (/)**: دریافت لیست کتاب‌ها از دیتابیس و ماژول Library و نمایش آن‌ها در رابط کاربری. همچنین نمایش پیام‌ها در صورت نبود کتاب.
- **افزودن کتاب (/add)**: دریافت داده از فرم HTML و ذخیره آن در دیتابیس و ماژول Library، سپس بازگشت به صفحه اصلی.
- **حذف کتاب (/delete)**: دریافت نام کتاب برای حذف، حذف آن از دیتابیس و ماژول Library، و بازگشت به صفحه اصلی.
این طراحی باعث می‌شود کاربر به راحتی بتواند مدیریت کتاب‌ها را انجام دهد و تمام تغییرات همزمان در دیتابیس و ماژول Library اعمال شود.

HTML و CSS:
رابط کاربری پروژه شامل فرم ثبت کتاب و لیست کتاب‌هاست. CSS صفحه را ساده و خوانا می‌کند و دکمه‌ها، فرم‌ها و لیست‌ها را به شکل زیبا نمایش می‌دهد. پیام‌ها به صورت واضح و قابل فهم در زمانی که هیچ کتابی موجود نیست نمایش داده می‌شوند. طراحی ساده باعث می‌شود کاربران بدون آموزش خاص بتوانند از برنامه استفاده کنند و تجربه کاربری خوبی داشته باشند.

نصب و اجرا:
- نصب پکیج‌ها: `pip install -r requirements.txt`
- اجرای برنامه: `python app.py`
- دسترسی از مرورگر: `http://localhost:5000`
این روش نصب و اجرا بسیار ساده است و نیاز به تنظیمات پیچیده ندارد. برنامه روی هر سیستمی که Python و pip داشته باشد اجرا خواهد شد.

نکات توسعه آینده:
- افزودن قابلیت ویرایش اطلاعات کتاب‌ها
- اضافه کردن جستجو و فیلتر پیشرفته بر اساس نام یا ژانر
- دسته‌بندی کتاب‌ها بر اساس ژانر، نویسنده یا تاریخ انتشار

- بهبود رابط کاربری و تجربه کاربری با افزودن طراحی ریسپانسیو و قابلیت استفاده روی موبایل
